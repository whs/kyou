แจกจ่าย

หลังจากสร้างหน้าเสร็จครบทุกหน้าแล้ว ใช้โปรแกรมแจกจ่ายในการส่งออกในรูปแบบที่ใช้งานง่าย

Output format
ในปัจจุบันระบบรองรับการส่งออกเพียงรูปแบบเดียว คือไฟล์ ZIP ที่มีไฟล์ที่อัพโหลดทั้งหมด และหน้าเอกสารทั้งหมดในรูปแบบ HTML โดยสามารถใช้บราว์เซอร์ที่ใช้ WebKit อ่านได้ (เช่น Google Chrome, Safari, Android Browser, Safari บน iOS)
App Icon
โลโก้หนังสือ
  • 16x16 ใช้ใน Chrome Web Store สำหรับ favicon (ไอคอนที่แท็บของ Chrome)
  • 48x48 ใช้ใน PhoneGap:Build โดยไม่เป็นการบังคับให้ใส่
  • 128x128 เป็นขนาดบังคับ โดยใช้ในไอคอนหน้าแท็บใหม่ของ Chrome และไอคอนโปรแกรมของ PhoneGap:Build
Application ID.
รหัสประจำโปรแกรม โดยมักใช้เป็นชื่อหนังสือในตัวอักษรอังกฤษและโดเมนเว็บไซต์จากหลังไปหน้า เช่นหากเว็บไซต์คือ sunburn.in.th และหนังสือชื่อ coreanime3 ให้กรอกว่า th.in.sunburn.coreanime3 โดยรหัสนี้จะต้องไม่ซ้ำกับใครทั่วโลก ค่าที่กรอกให้ตอนเริ่มต้นไม่ควรใช้

หากไม่มีโดเมนของตัวเอง ให้ใส่อะไรก็ได้ที่คล้ายโดเมน เช่น yourname.coreanime3

Application version
เลขรุ่นการส่งออกครั้งนี้ โดย Chrome Web Store จะไม่รับการอัพเดตที่มีเลขรุ่นเท่ากับ หรือน้อยกว่ารุ่นปัจจุบันที่อัพโหลดไว้เดิม ตัวอย่างการกรอก เช่น 1, 1.0 หรือ 1.0.0
Description
อธิบายหนังสือ โดยจะใช้ในข้อมูลประกอบ PhoneGap:Build data และในหน้าจัดการส่วนขยายของ Chrome
Google Analytics ID
กรอกเพื่อติดตามการเปิดทุกหน้าโดยใช้ Google Analytics ID ที่ระบุไว้ เช่น UA-27038379-9 (ไม่บังคับกรอก)

การใช้ไฟล์ส่งออก

หลังจากส่งออกแล้ว จะได้ไฟล์ ZIP ที่มีไฟล์ HTML พร้อมไฟล์ทรัพยากรที่อัพโหลดไว้
  • แตกไฟล์แล้วอัพโหลดเนื้อหาไปยังเว็บโฮสต์ติ้งทั่วไป แล้วแจกจ่ายลิงก์
  • อัพโหลดไฟล์ ZIP ไปที่ Chrome Web Store โดยในครั้งแรกจะต้องจ่ายค่าแรกเข้า $5 ก่อน ผู้ใช้ที่ใช้ Google Chrome สามารถดาวน์โหลดหนังสือไปอ่านออฟไลน์ได้จาก Chrome Web Store
  • อัพโหลดไฟล์ ZIP ไปที่บริการ PhoneGap:Build (ใช้ตัวเลือกแบบ private) จะได้ไฟล์โปรแกรมสำหรับ Android, Windows Phone, BlackBerry, webOS และ Symbian. (iOS จะต้องตั้งค่าเพิ่มเติม) โดยบัญชีแบบฟรีสามารถใช้โปรแกรมแบบ private ได้เพียงโปรแกรมเดียวเท่านั้น
