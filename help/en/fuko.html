Distribution

After finishing all pages, use the Distribution application to export into easy-to-use formats.

Output format
Only one format is supported at this moment which is a ZIP file that contains all uploaded files and all pages in HTML format. It can be read by WebKit-based browsers (such as Google Chrome, Safari, Android Browser, iOS's Safari)
App Icon
Logo of the book.
  • 16x16 is used in Chrome Web Store for favicon (Chrome's tab icon)
  • 48x48 is used in PhoneGap:Build. This is not required
  • 128x128 is the required size. It is used in Chrome's new tab page icon and PhoneGap:Build app icon.
Application ID.
ID of the application. Usually this is your book name in alphanumeric and your website in reverse order, for example if your website is sunburn.in.th and your book is called coreanime3 enter th.in.sunburn.coreanime3. This ID must be globally unique. The default value should not be used.

If you don't have your own domain, enter something that sounds like domain name like yourname.coreanime3 is okay.

Application version
Version of this build. Chrome Web Store will not accept new updates that have the same or lower version number as the current uploaded build. Examples of valid value are: 1, 1.0 or 1.0.0
Description
Describe your book. This is used in PhoneGap:Build data and Chrome's extension management page.
Google Analytics ID
Optional, if filled it will track every page views with specified Google Analytics ID. Example of valid value is: UA-27038379-9

Using the exported file

After exporting, you will get a ZIP file that contains HTML files and your uploaded resources.
  • Extract the ZIP and upload its content to any web hosting and share the link.
  • Upload the ZIP to Chrome Web Store. Note that CWS requires $5 one time payment for app submission. Users using Google Chrome then can download the book to read offline from the Chrome Web Store
  • Upload the ZIP to PhoneGap:Build service. (Use the private option) You will get standalone application files for Android, Windows Phone, BlackBerry, webOS and Symbian. (iOS build requires additional setup) Note that free account can only build one private apps.
